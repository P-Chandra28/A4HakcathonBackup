Project Overview

Understanding the machine learning challenge and our approach to solving hospital re-admission prediction.

Problem Statement

Hospital readmissions are one of the costliest challenges facing healthcare systems, but conventional models fail to predict readmissions well. Many existing models use exclusively manually-engineered features, which are dataset-specific and lack the flexibility to adapt to different healthcare environments.

Key Challenges
  • High costs associated with preventable readmissions
  • Poor prediction accuracy of existing models
  • Dataset-specific feature engineering limitations
  • Lack of real-time risk assessment capabilities

Our Solution

Building a predictive model that accurately classifies patients as readmitted or not, helping healthcare providers identify high-risk patients and take proactive measures.

Machine Learning Task
Task: Classification
Project Objective

Build a predictive model that can accurately classify patients as readmitted or not, based on the provided features. This model helps healthcare providers identify high-risk patients and take proactive measures to prevent readmissions.

Dataset & Data Preprocessing

Dataset Features

  • age: Patient's age range.
  • time_in_hospital: Patient's stay duration.
  • n_lab_procedures: Number of lab procedures.
  • n_procedures: Number of non-lab procedures.
  • n_medications: Number of medications.
  • n_outpatient: Outpatient visits.
  • n_inpatient: Inpatient visits.
  • n_emergency: Emergency visits.
  • medical_specialty: Admitting physician's specialty.
  • A1Ctest/glucose_test: Test results.
  • change: Change in diabetes medication.
  • diabetes_med: Diabetes medication prescribed.
  • diag_1/2/3: Primary/secondary/tertiary diagnoses.
  • readmitted: Target variable (yes/no).

Project Journey

The key steps we took to develop our predictive model and the challenges we addressed along the way.

Our Approach

  • Data Preprocessing

    Analyzed the dataset through EDA to identify missing values and handled them with KNN imputation while simplifying categories and dropping irrelevant columns.

  • Feature Engineering

    Created new features, applied one-hot encoding, scaling with StandardScaler, and addressed class imbalance using SMOTENN to improve model performance.

  • Model Training

    Trained multiple models and finally built a stacking ensemble (Logistic Regression, Random Forest, AdaBoost), validated with Stratified K-Fold and optimized thresholds for better F1-score.

  • Modular Coding

    Entire pipeline was developed with a modular coding approach, ensuring reusability, clarity, and ease of collaboration.

  • Model Deployment

    Final model was deployed using Flask, providing an interface for real-time hospital readmission risk prediction.

Challenges Addressed

  • Missing Value Imputation

    Key diagnostic and specialty columns had many missing values, which we handled using KNN imputation for more robust estimates.

  • Model Performance & Feature Bias

    Initial models struggled with low accuracy (~60%) due to outliers and feature bias, so we used stacking with cross-validation to improve stability.

  • Class Imbalance

    The dataset was highly imbalanced, and we resolved this using balancing techniques to ensure reliable predictions for both readmission and non-readmission cases.

Performance Metrics

  • Average F1-Score: 0.83
  • Average ROC-AUC: 0.87
  • Average Accuracy: 0.79
